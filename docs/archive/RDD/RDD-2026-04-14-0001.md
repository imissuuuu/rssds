# RDD - 2026-04-14-0001
**Project Name:** Phase 6 - ステータスバー + 設定画面

## 1. 機能一覧

**機能要件:**
- FR-1: 上画面最上部にステータスバーを全画面共通で常時表示（バッテリー残量・充電中インジケータ・現在時刻）
- FR-2: フィードリスト末尾に `Settings` エントリを追加し、A ボタンで設定画面へ遷移
- FR-3: 設定画面（上下で項目選択、左右/A で値変更、B で戻る）
- FR-4: 十字キー感度調整（Key Repeat Delay / Key Repeat Interval）を設定項目として提供
- FR-5: 設定を `sdmc:/3ds/rssreader/settings.json` に永続化（起動時ロード、不在時はデフォルト値）

**非機能要件:**
- NFR-1: `Screen` enum に `Settings` を追加して既存の画面管理に統合
- NFR-2: キー感度設定は `main.cpp` の `hidSetRepeatParameters` 呼び出し箇所に適用

## 2. 詳細仕様

### FR-1: ステータスバー
- **表示位置**: 上画面（400×240px）の最上部、高さ 14px 程度の帯
- **表示内容**:
  - 左端: 現在時刻（`HH:MM` 形式、`osGetTime()` + `localtime` で取得）
  - 右端: バッテリー残量（`PTMU_GetBatteryLevel` で取得、0〜5 段階 → %換算）+ 充電中は `[CHG]` 表示
- **既存コンテンツへの影響**: ステータスバー分（14px）だけ各画面のコンテンツ開始 Y 座標をオフセット

### FR-2: Settings エントリ
- フィードリストの通常エントリと同じスタイルで描画
- インデックスは feedConfigs の末尾 + 1（仮想エントリ）
- 選択・A 押下で `Screen::Settings` へ遷移

### FR-3: 設定画面
- **上画面**: "Settings" タイトル
- **下画面**: 設定項目リスト
  - 選択行はハイライト表示
  - 各項目: `[項目名]   [< 現在値 >]` のフォーマット
  - 左右キーで値を変更（即時反映）、B で前の画面（FeedList）へ戻る

### FR-4: 十字キー感度
- **対象**: ArticleView の上下スクロール（Up/Down キーのリピート）のみ
- フィードリスト・記事一覧のナビゲーションは対象外

| 設定項目 | 選択肢（ms） | デフォルト |
|----------|-------------|-----------|
| Key Repeat Delay | 200 / 300 / 400 / 500 | 300 |
| Key Repeat Interval | 50 / 80 / 120 / 160 | 80 |

- 値変更時に即時反映（ArticleView 遷移時に `hidSetRepeatParameters` を設定値で呼び出す）

### FR-5: 永続化
- **保存形式**: JSON（`nlohmann/json` を使用）
- **フィールド**:
  ```json
  {
    "key_repeat_delay_ms": 300,
    "key_repeat_interval_ms": 80
  }
  ```
- 起動時（`main.cpp` 初期化時）にロード → `AppState` に保持
- 設定変更時（値変更のたびに or B ボタンで戻るタイミングで）書き込み

## 3. UI/UX考慮事項

```
+------------------------------------------+

+------------------------------------------+
```

### 設定画面（下画面）
```
+--------------------------------+
| Key Repeat Delay   < 300 ms >  |  ← 選択中（ハイライト）
| Key Repeat Interval  80 ms     |
|                                |
|                                |
| Left/Right:change  B:back      |  ← 操作ガイド
+--------------------------------+
```

### ボタン操作
| ボタン | 動作 |
|--------|------|
| Up/Down | 設定項目選択 |
| Left/Right | 値変更 |
| B | 設定画面を閉じて FeedList へ戻る |

## 4. 追加設計情報

### 制約
- `PTMU_GetBatteryLevel` は PTMU サービスの初期化（`ptmuInit()`）が必要
- `osGetTime()` はミリ秒精度の epoch 時刻を返す → `localtime` で HH:MM に変換
- バッテリー取得は毎フレームではなく一定間隔（1秒ごと等）にしてオーバーヘッドを抑える

### 既存コードへの影響
- `source/ui.h`: `Screen` enum に `Settings` 追加、`AppState` に設定値フィールド追加
- `source/ui.cpp`: ステータスバー描画を各画面に追加、設定画面の描画・入力処理を追加
- `source/main.cpp`: `hidSetRepeatParameters` 呼び出しを設定値ベースに変更、settings.json ロード追加
- 新規ファイル `source/settings.cpp / .h`: 設定の読み書きロジック
