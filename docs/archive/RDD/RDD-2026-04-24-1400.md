# RDD - 2026-04-24-1400
**Project Name:** レビューシステム本格化

## 1. 機能一覧

### 機能要件

- **FR-1** GitHub Actions CI の導入（PR → main トリガー）
- **FR-2** CI: devkitpro/devkitarm Docker によるビルド確認（`make` + 成果物検証）
- **FR-3** CI: clang-format によるフォーマットチェックゲート
- **FR-4** CI: cppcheck による静的解析（`source/` 対象）
- **FR-5** CI: clang-tidy の導入試行（ARM フラグ除去が解決できれば必須ゲート化、困難なら後フェーズ）
- **FR-6** ローカル: clang-format チェックを commit 前フローに追加
- **FR-7** ゼロワーニング原則: Makefile に `-Wextra` 追加
- **FR-8** ゼロワーニング原則: CI のみ `-Werror` を適用（PR ゲート）

### 非機能要件

- **NFR-1** Docker タグは `latest` でなく日付タグ固定（再現性確保）
- **NFR-2** CI の必須ゲートが落ちた場合、main へのマージをブロックする
- **NFR-3** CodeQL は任意・non-blocking ジョブとして追加（週次または PR 補助）
- **NFR-4** ローカルの開発フロー（wf）は現状を維持し、余計な自動化で阻害しない
- **NFR-5** clang-tidy が困難な場合でも FR-2〜FR-4 + FR-7〜FR-8 の達成をもって Phase 14 完了とする

## 2. 詳細仕様

### FR-1〜FR-4: GitHub Actions ワークフロー

**トリガー:** `pull_request`（対象ブランチ: `main`）、`workflow_dispatch`

**ジョブ構成:**

```
job: format
  - checkout
  - clang-format インストール（apt-get）
  - git ls-files で *.cpp/*.h を列挙
  - clang-format --dry-run --Werror -style=file

job: build
  container: devkitpro/devkitarm:<日付タグ>
  - checkout（submodules: recursive）
  - make -j$(nproc)
  - test -f rssds.3dsx（成果物検証）
  - upload-artifact（*.3dsx, *.elf）

job: static-analysis
  - checkout
  - cppcheck インストール
  - cppcheck --enable=all --language=c++ --platform=unix32
              --suppress=missingIncludeSystem
              source/
```

**必須ゲート:** format / build / static-analysis の全ジョブ pass が main マージ条件

### FR-5: clang-tidy 導入試行

**手順:**
1. CI 内で `bear -- make` を実行して `compile_commands.json` を生成
2. ARM 固有フラグ（`-mword-relocations`, `-specs=3dsx.specs`, `-march=armv6k` 等）を除去するスクリプトまたは `.clang-tidy` の `RemovedArgs` で対処
3. `HeaderFilterRegex: "source/|include/"` で SDK ヘッダノイズを除外
4. clang-tidy を `source/` に対して実行

**判定:** CI で安定動作すれば必須ゲートに昇格。不安定・設定工数過多なら non-blocking または後フェーズへ。

### FR-6: ローカル clang-format

- `.clang-format` ファイルをリポジトリに追加（スタイル定義）
- `commit-dev.sh` または pre-commit hook で `clang-format --dry-run --Werror` を実行
- 違反があればコミットをブロックし、差分を表示する

### FR-7〜FR-8: ゼロワーニング

- `Makefile` の `CFLAGS` に `-Wextra` を追加（既存 `-Wall` に追記）
- CI の build ジョブの `make` コマンドに `CFLAGS_EXTRA=-Werror` を渡すか、CI 専用 Makefile target を用意
- ローカルには `-Werror` を適用しない（開発速度を阻害しないため）

### NFR-3: CodeQL（補助）

- GitHub デフォルト設定（`none` build mode）で週次スケジュール実行
- merge blocker にしない
- アラートが継続的に有用な場合のみ昇格を検討

## 3. UI/UX 考慮事項

該当なし（開発インフラ整備タスク）。

## 4. 追加設計情報

### 制約・前提

- devkitPro ツールチェーンは Windows ローカルと Linux Docker で動作環境が異なる（パスが異なる）
- cppcheck の `-I` フラグは Docker 内のパス（`/opt/devkitpro/libctru/include` 等）を使う必要がある
- clang-tidy は GCC 系クロスコンパイラ固有フラグとの摩擦があるため、段階的導入とする
- 現在の Makefile は devkitPro 提供テンプレートベース。`-Wextra` 追加時に既存コードで新規警告が出る可能性があり、追加前に一度ビルドして警告数を確認する

### 既存コードへの影響

- `Makefile`: `CFLAGS` に `-Wextra` 追加
- `commit-dev.sh` または新規 pre-commit hook: clang-format チェック追加
- `.github/workflows/ci.yml`: 新規作成
- `.clang-format`: 新規作成（スタイル定義）
- `.clang-tidy`: 新規作成（試行時）

### フロー全体像（構想ファイルより）

```
ローカル開発
  → clang-format チェック（新規）
  → commit（dev ブランチ）
  → PR to main
  → GitHub Actions（build + format + cppcheck [+ clang-tidy]）
  → 修正ループ
  → main マージ
```
