# RDD - 2026-04-19-1200
**Project Name:** Phase 7 画像表示 MVP

## 1. 機能一覧

### 機能要件
- **FR-1**: 記事本文（Readability が本文判定した領域）内の `<img>` タグを抽出する
- **FR-2**: 抽出した画像 URL を libcurl で逐次ダウンロードする（HTTPS 対応）
- **FR-3**: JPEG / PNG を `stb_image` でデコードする
- **FR-4**: 上画面幅（400px）に合わせてアスペクト比維持で縮小する（元が小さい場合は原寸）
- **FR-5**: デコード結果を citro2d テクスチャに変換して記事本文末尾に「画像」セクションとして表示する
- **FR-6**: 本文は即座に表示し、画像エリアはバックグラウンド取得中「取得中...」と表示する
- **FR-7**: ダウンロード/デコード失敗時は「[画像取得失敗]」プレースホルダを表示
- **FR-8**: 1記事あたり最大 10 枚まで対象（超過分は無視）
- **FR-9**: 1画像あたり最大 2MB まで対象（Content-Length 超過時はスキップ）

### 非機能要件
- **NFR-1**: Old 3DS でも動作すること（RAM 64MB 制約下でテクスチャ保持数を抑制）
- **NFR-2**: メモリリーク無し（記事遷移時に前記事の画像テクスチャを解放）
- **NFR-3**: 画像取得失敗・サイズ超過・デコード失敗が起きても他の画像や本文表示に影響しない
- **NFR-4**: 画像取得は blocking だが本文閲覧は取得中でも可能（進捗非同期ではなく、本文表示後に裏で順次取得する設計）

## 2. 詳細仕様

### 2.1 画像抽出
- **入力**: Readability が出力する本文 DOM ツリー
- **処理**: 本文判定されたノード配下の `<img src="...">` を順に収集。`src` が相対URLなら記事URL基準で解決
- **出力**: `std::vector<std::string> imageUrls`（最大10件）

### 2.2 ダウンロード
- **入力**: 画像 URL
- **処理**:
  1. HEAD リクエストで Content-Length 確認（>2MB なら即スキップ）
  2. 取得できない場合は GET で受信しつつサイズ監視、超過時は中断
  3. 受信データをメモリバッファに保持
- **出力**: `std::vector<uint8_t>` または失敗フラグ

### 2.3 デコード
- **入力**: 画像バイナリバッファ
- **処理**: `stbi_load_from_memory()` で RGBA8 へデコード
- **出力**: 幅・高さ・RGBA ピクセル配列

### 2.4 スケーリング・テクスチャ化
- **入力**: RGBA ピクセル配列
- **処理**:
  1. 幅が 400px を超える場合、400px に合わせて縮小（アスペクト比維持、最近傍またはバイリニア）
  2. citro2d テクスチャサイズは POT（power-of-two）必要 → 次の2の冪にパディング、有効領域のみ描画
  3. RGBA → 3DS 固有のタイル形式へ swizzle
- **出力**: `C3D_Tex` または `C2D_Image`

### 2.5 描画
- 記事本文の `wrapText` 済み行列の末尾に「画像」セクションを追加
- 各画像を縦に並べて描画（画像間に数px のマージン）
- ページスクロール時に画像の Y 座標を計算に含める

### 2.6 ライフサイクル
- **記事選択時**: 画像URL抽出 → 本文表示開始 → バックグラウンドで画像取得・デコード・テクスチャ化を逐次実行（完了したものから描画更新）
- **記事遷移時**: 全テクスチャ解放（C3D_TexDelete）

## 4. 追加設計情報

### 4.1 実装上の制約
- `stb_image` を `lib/stb/` に追加（ヘッダオンリーなので Makefile 対応のみ）
- libcurl 進捗コールバックで Content-Length 監視（サイズ超過中断用）
- `C3D_TexInit` / `C3D_TexDelete` のタイミングに注意（GPU コマンドリスト実行中に delete しない）
- Old 3DS のメモリ制約下では同時保持テクスチャ数に実運用上の上限あり（10枚 × 400×240 RGBA ≈ 3.7MB + POT パディングで 4〜6MB 程度、許容範囲）

### 4.2 既存コードへの影響（概要のみ、詳細は DESIGN で記述）
- `source/readability.cpp` — `<img>` タグの URL 抽出を追加（現行は SKIP_SUBTREE 対象の可能性あり、要確認）
- `source/ui.cpp` / `source/ui.h` — ArticleView に画像セクション描画を追加
- `source/` 新規ファイル（想定）: `image_loader.cpp/h`（ダウンロード＋デコード＋テクスチャ化）
- `Makefile` — `lib/stb/` のインクルードパス追加
- `HANDOFF/context_handoff.md` — Phase 7 完了後の状態反映

### 4.3 スコープ外（将来フェーズへ）
