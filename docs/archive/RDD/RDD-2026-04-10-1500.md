# RDD - 2026-04-10-1500
**Project Name:** 本文取得の安定化

## 1. 機能一覧

### 機能要件
- **FR-1**: 本文抽出精度の向上 — Lexbor でDOMを構築し、Readability系ヒューリスティクスで本文ブロックを選択して抽出すること
- **FR-2**: HN型フィード対応 — フィード本文が空またはN文字未満の場合、リンク先URLから本文をフェッチして表示すること
- **FR-3**: Lexbor 導入 — アマルガメーション（単一Cファイル）形式でプロジェクトに組み込み、devkitProでビルドできること

### 非機能要件
- **NFR-1**: 3DS メモリ制約 — 本文フェッチ・抽出処理のメモリ使用量が既存 MVP と同等以下であること
- **NFR-2**: 失敗時の graceful degradation — HTTP失敗（タイムアウト/TLS/リダイレクト/403）・抽出失敗のいずれでもフィードの既存テキストを表示してクラッシュしないこと

## 2. 詳細仕様

### FR-1: 本文抽出精度の向上
- **現状**: `html_strip.cpp` が `<script>`/`<style>` 除去＋テキスト化のみ
- **入力**: 取得した HTML 文字列
- **処理（Readability系軽量版）**:
  1. 前処理: `script`/`style`/`noscript`/`nav`/`header`/`footer`/`aside`/`form` を破棄
  2. 候補収集: `article`/`main` があれば優先候補。なければ `div`/`section` を候補に `p`/`pre`/`li` のテキスト量を集計
  3. スコアリング: `テキスト長 + 句読点数` を加算し、`× (1 - リンク密度)` で補正
  4. 勝者の前後兄弟で条件を満たすブロックを取り込む
- **出力**: UTF-8 プレーンテキスト（既存表示パイプラインに渡す）

### FR-2: HN型フィード対応
- **判定条件**: フィードの `<description>` または `<summary>` が空、または N文字未満（閾値は実装時に決定）
- **入力**: フィード記事の `<link>` URL
- **処理**: URL から HTML をフェッチ → FR-1 の抽出処理を適用
- **出力**: 抽出テキストを記事本文として表示
- **失敗時**: フィードの既存テキスト（短くても）を表示。元テキストを上書きしない

### FR-3: Lexbor 導入
- **採用理由**: 依存なし・C実装（例外不要）・アマルガメーション対応・ARM64実績あり（v2.7.0以降前提）
- **組み込み方法**: ホスト環境でアマルガメーションを生成 → 単一Cソースを `lib/lexbor/` に配置、Makefile に追記
- **代替**: Lexborのビルドが困難な場合、gumbo-parser（移管後系、Apache-2.0）を使って抽出ロジックを完全自前実装
- **調査資料**: `.claude/research/html-extraction-research.md`

## 3. UI/UX考慮事項

既存の記事本文表示画面をそのまま使用。ユーザーから見た変化は「本文が表示されるサイトが増える」のみ。

失敗時の表示：
```
[記事タイトル]

（本文を取得できませんでした）
[フィードの短い説明文があればそれを表示]
```
