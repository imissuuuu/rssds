# RDD - 2026-04-10-1600
**Project Name:** Phase 0 — アーキテクチャ基盤

## 1. 機能一覧

### 機能要件
- **FR-1**: オンデマンドロード — 起動時はフィード一覧のみ表示し、選択時にそのフィードを取得する
- **FR-2**: テキスト描画基盤 — 画面サイズに基づいて折り返し・フォントサイズを計算し、将来のスタイル対応（見出し・本文・絵文字）を追加できる設計にすること
- **FR-3**: フィード設定 JSON 化 — feeds.txt をJSON形式に移行し、フィード単位の設定（表示名・全文取得ON/OFF等）を持てること

### 非機能要件
- **NFR-1**: スレッド不使用 — ネットワーク処理はすべて同期・オンデマンドで行い、バックグラウンドスレッドを使わないこと
- **NFR-2**: 将来拡張性 — UIロジックとデータロジックを分離し、画面サイズの決め打ち数値を一箇所に集約すること

## 2. 詳細仕様

### FR-1: オンデマンドロード
- **現状**: 起動時に全フィードを一括フェッチ → 画面が長時間真っ暗
- **新フロー**:
  ```
  起動 → feeds.json を読む（ローカルのみ、即座）
       → フィード一覧を表示
       → フィード選択 → HTTP フェッチ＋パース → 記事一覧表示
       → 記事選択 → 本文フェッチ → 本文表示
  ```
- フェッチ中は「Loading...」等のインジケータを表示すること
- 各フェッチは独立して失敗できること（1フィードが失敗しても他に影響しない）

### FR-2: テキスト描画基盤
- **画面サイズ定数を一箇所に集約**:
  - 上画面: 400×240
  - 下画面: 320×240
- **折り返し計算**: フォントサイズとマージンから1行あたりの文字数を動的に計算（ハードコード禁止）
- **テキストスタイル型を定義**: 最低限 `BODY`・`HEADING` の2種類を持ち、将来追加できる構造
  - `BODY`: 標準サイズ
  - `HEADING`: 標準より大きいサイズ（係数で定義）
- UTF-8 バイト境界での折り返しは既存実装を流用

### FR-3: フィード設定 JSON 化
- **feeds.txt を廃止**し `feeds.json` に移行
- **SDカード配置**: `sdmc:/3ds/rssreader/feeds.json`
- **最小スキーマ**:
  ```json
  {
    "feeds": [
      {
        "url": "https://example.com/feed.xml",
        "name": "Example Feed",
        "fetch_full_text": false
      }
    ]
  }
  ```
- `name` 省略時はURLをそのまま表示
- `fetch_full_text` 省略時は `false`
- **JSONパーサ**: `nlohmann/json`（単一ヘッダ、例外無効オプションあり）を `lib/` に追加
- 後方互換は不要（MVPはまだ未リリース）

## 4. 追加設計情報

### 制約
- スレッド不使用（クラッシュリスク）
- C++ 例外不可（devkitPro）
- `nlohmann/json` は `-fno-exceptions` 対応を確認してから採用
- → 詳細は `HANDOFF/context_handoff.md` 参照

### 既存コードへの影響
- `source/main.cpp` — 起動フロー大幅変更
- `source/ui.cpp` / `source/ui.h` — テキスト描画基盤の刷新、スタイル型追加
- `source/feed_config.cpp` / `source/feed_config.h` — feeds.txt → JSON 化
- `source/net.cpp` — 変更なし（フェッチは既存流用）
- `lib/` — `nlohmann/json.hpp` 追加

### 将来を見越した設計指針
- 画面サイズ・フォントサイズはすべて定数/計算経由（ハードコード禁止）
- テキストスタイルは列挙型で管理し、追加しやすくしておく
- フィード設定スキーマは `feeds.json` のバージョン番号を持つことを将来的に検討
