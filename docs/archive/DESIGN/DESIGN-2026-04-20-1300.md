## 1. 全体方針

### 核心アイデア: センチネルマーカー方式

`<img src="URL">` を見つけたとき、プレーンテキスト本文の中に `\x01URL\x01`（ASCII SOH = 0x01）を埋め込む。この値は HTML・URL・日本語テキストに実在しないため安全。

マーカーは2か所で生成される：
- **`readability.cpp`**: `serialize_cb` で lexbor DOM を走査中に `<img>` を検出 → マーカーを本文文字列に append
- **`html_strip.cpp`**: `stripHtml` で `<img src=...>` を検出 → マーカーを emit（RSS 直接コンテンツ用）

`drawArticleView` では `stripHtml(art.content)` の後（マーカーはそのまま通過）、`parseContentLines` でテキスト行と画像行に分割してレンダリングする。

既存の「画像セクション（テキスト末尾に縦積み）」は本フェーズで**廃止**し、インライン方式に置き換える。

### 3.2 `source/html_strip.cpp`

`stripHtml` 内の `<` ハンドリングブロック（line 95〜143）に `<img>` 特別処理を追加。

追加箇所: `isSkipContentTag` チェックの直後、`isBlockTag` チェックの前に挿入。

**追加コード:**
```cpp
// img タグ: src= を抽出して \x01URL\x01 マーカーを emit（絶対URLのみ）
if (!isClose && nameLen == 3 && strncasecmp(tagStart, "img", 3) == 0) {
    // タグ末尾 '>' を探す
    const char* tagEnd = p;
    while (tagEnd < end && *tagEnd != '>') tagEnd++;
    // src= 属性を簡易パース
    const char* q = nameEnd;
    while (q < tagEnd - 3) {
        if (strncasecmp(q, "src=", 4) == 0) {
            q += 4;
            char quote = (*q == '"' || *q == '\'') ? *q++ : 0;
            const char* valStart = q;
            if (quote) { while (q < tagEnd && *q != quote) q++; }
            else        { while (q < tagEnd && !isspace((unsigned char)*q) && *q != '>') q++; }
            std::string src(valStart, q - valStart);
            // 絶対 HTTP(S) URL のみ emit（data: や相対は除外）
            if ((src.compare(0,7,"http://") == 0 || src.compare(0,8,"https://") == 0)
                    && src.compare(0,5,"data:") != 0) {
                result += '\x01';
                result += src;
                result += '\x01';
            }
            break;
        }
        q++;
    }
    p = tagEnd;
    if (p < end) p++;
    continue;
}
```

また、`\x01` は `<` でも `&` でもないため、既存の通常文字処理 `result += *p++` でそのまま通過する。追加の変更不要。

#### (b) 新関数 `parseContentLines`（`wrapText` の後ろに追加）

```cpp
static std::vector<ContentLine> parseContentLines(const std::string& body, int maxPx) {
    std::vector<ContentLine> result;
    size_t i = 0, n = body.size();
    while (i <= n) {
        size_t m = body.find('\x01', i);
        if (m == std::string::npos) m = n;

        // テキスト区間 [i, m)
        if (m > i) {
            std::string seg = body.substr(i, m - i);
            for (auto& tl : wrapText(seg, maxPx)) {
                ContentLine cl;
                cl.kind = LineKind::Text;
                cl.text = std::move(tl);
                result.push_back(std::move(cl));
            }
        }
        if (m >= n) break;

        // 画像マーカー: 閉じ \x01 を探す
        size_t urlStart = m + 1;
        size_t urlEnd   = body.find('\x01', urlStart);
        if (urlEnd == std::string::npos) { i = urlStart; continue; }
        std::string url = body.substr(urlStart, urlEnd - urlStart);
        if (!url.empty()) {
            ContentLine cl;
            cl.kind     = LineKind::Image;
            cl.imageUrl = std::move(url);
            result.push_back(std::move(cl));
        }
        i = urlEnd + 1;
    }
    return result;
}
```

## 4. 各機能の詳細設計

### FR-2: 自動インラインロード の非同期性

`imgCache.tick()` は毎フレーム呼ばれ、`visible` セットにある URL を自動 submit する。ImageLoader のワーカースレッドが実際のダウンロードを担当（既存の LightLock + libctru thread）。UIスレッドはブロックしない。

### `\x01` マーカーの耐性

- `stripHtml` 内: `<` でも `&` でもない文字は `result += *p++` でそのまま通過 → マーカー保持
- `wrapText` 内: UTF-8 デコードで `\x01`（< 0x80）は ASCII 1文字として扱われる
  - ただし `parseContentLines` を経由するため、`wrapText` には `\x01` が渡らない ✓

### `IMG_INLINE_H = 192` の根拠

- LINE_HEIGHT = 16px → 192 = 12行分（整数倍、スクロール計算がきれいになる）
- 3DS 上画面 TOP_H = 240px → 240 - STATUSBAR_H(14) - TOP_CONTENT_Y付近 ≒ 220px
- 1枚表示しつつ前後テキストも見える高さとして適切
