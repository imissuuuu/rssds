## 1. 全体方針

- **Lexbor amalgamation を `lib/lexbor/` に配置**し、Makefile の自動検出で取り込む（手動 `.o` 追記不要）
- **本文抽出ロジックは `html_strip.cpp` に集約**する。`extractContent()` を新規追加し、`article.cpp` からはこれを呼ぶ形に整理する
- `stripHtml()` は RSS description の表示整形に引き続き使用。変更しない
- **FR-2（HN型フォールバック）は `ui.cpp` に既実装済み**（`CONTENT_SHORT_THRESHOLD = 200`、`fetchArticleBody()` 呼び出し）。`article.cpp` 内部が Lexbor に切り替わることで品質が向上するため、`ui.cpp` の変更は不要
- 例外不可制約（devkitPro）のため、すべての Lexbor 戻り値を `lxb_status_t` で確認し、エラー時は `stripHtml(html)` にフォールバックする

## 2. 新規ファイル

### `lib/lexbor/lexbor.c` + `lib/lexbor/lexbor/` ディレクトリ

Lexbor v3.0.0 のアマルガメーション生成物。ホスト環境で以下を実行して生成する:

```bash
# Lexbor リポジトリをクローン
git clone https://github.com/lexbor/lexbor --branch v3.0.0 --depth 1
cd lexbor
python utils/amalgamation/build.py
# 出力: build/amalgamation/lexbor.c, build/amalgamation/lexbor/lexbor.h (他ヘッダ群)
```

配置後のディレクトリ構造:
```
lib/lexbor/
  lexbor.c            ← 単一実装ファイル（Makefile が自動検出）
  lexbor/
    lexbor.h          ← メインヘッダ（#include "lexbor/lexbor.h" でインクルード）
    core/
    html/
    dom/
    ... （その他ヘッダ群）
```

インクルード方法: `#include "lexbor/lexbor.h"` （INCLUDES に `lib/lexbor` を追加後）

### `source/html_strip.cpp`

末尾に以下を追加（既存コードは変更なし）:

```cpp
#include "lexbor/lexbor.h"

// ---- 内部定数 ----

static const char* NOISE_TAGS[] = {
    "script","style","noscript","nav","header","footer","aside","form", nullptr
};
static const char* PRIORITY_TAGS[] = { "article", "main", nullptr };
static const char* CANDIDATE_TAGS[] = { "div", "section", nullptr };

// ---- 内部ヘルパー ----

// node のタグ名が name と一致するか（大文字小文字無視）
static bool tagEq(lxb_dom_node_t* node, const char* name) {
    if (node->type != LXB_DOM_NODE_TYPE_ELEMENT) return false;
    size_t len;
    const lxb_char_t* tag = lxb_dom_element_local_name(
        lxb_dom_interface_element(node), &len);
    if (!tag) return false;
    for (size_t i = 0; i < len && name[i]; ++i)
        if (tolower((unsigned char)tag[i]) != name[i]) return false;
    return name[len] == '\0';
}

// NOISE_TAGS に一致するノードを DOM から除去する（再帰）
static void removeNoiseNodes(lxb_dom_node_t* root) {
    lxb_dom_node_t* node = lxb_dom_node_first_child(root);
    while (node) {
        lxb_dom_node_t* next = lxb_dom_node_next(node);
        for (int i = 0; NOISE_TAGS[i]; ++i) {
            if (tagEq(node, NOISE_TAGS[i])) {
                lxb_dom_node_remove(node);
                goto next_node;
            }
        }
        removeNoiseNodes(node);
        next_node:
        node = next;
    }
}

// PRIORITY_TAGS（article/main）を幅優先探索で返す。なければ nullptr
static lxb_dom_node_t* findPriorityNode(lxb_dom_node_t* root) {
    lxb_dom_node_t* node = lxb_dom_node_first_child(root);
    while (node) {
        for (int i = 0; PRIORITY_TAGS[i]; ++i)
            if (tagEq(node, PRIORITY_TAGS[i])) return node;
        lxb_dom_node_t* found = findPriorityNode(node);
        if (found) return found;
        node = lxb_dom_node_next(node);
    }
    return nullptr;
}

// node 以下のリンクテキスト長を返す（link_density 計算用）
static size_t linkTextLen(lxb_dom_node_t* node) {
    size_t total = 0;
    lxb_dom_node_t* child = lxb_dom_node_first_child(node);
    while (child) {
        if (tagEq(child, "a")) {
            size_t len;
            lxb_char_t* txt = lxb_dom_node_text_content(child, &len);
            if (txt) { total += len; lexbor_free(txt); }
        } else {
            total += linkTextLen(child);
        }
        child = lxb_dom_node_next(child);
    }
    return total;
}

// p/pre/li の文字数と句読点数の合計を返す
static double scoreNode(lxb_dom_node_t* node) {
    static const char* TEXT_TAGS[] = { "p", "pre", "li", nullptr };
    double score = 0.0;
    lxb_dom_node_t* child = lxb_dom_node_first_child(node);
    while (child) {
        for (int i = 0; TEXT_TAGS[i]; ++i) {
            if (tagEq(child, TEXT_TAGS[i])) {
                size_t len;
                lxb_char_t* txt = lxb_dom_node_text_content(child, &len);
                if (txt) {
                    score += (double)len;
                    for (size_t j = 0; j < len; ++j) {
                        char c = (char)txt[j];
                        if (c == '.' || c == ',' || c == ';' || c == ':' ||
                            c == '!' || c == '?') score += 1.0;
                    }
                    lexbor_free(txt);
                }
                break;
            }
        }
        score += scoreNode(child);
        child = lxb_dom_node_next(child);
    }
    return score;
}

// node 以下のテキストをプレーンテキスト化する（ブロックタグで改行）
static void nodeToText(lxb_dom_node_t* node, std::string& out) {
    static const char* BLOCK[] = {
        "p","div","br","li","h1","h2","h3","h4","h5","h6",
        "tr","blockquote","pre", nullptr
    };
    if (node->type == LXB_DOM_NODE_TYPE_TEXT) {
        size_t len;
        lxb_char_t* txt = lxb_dom_node_text_content(node, &len);
        if (txt) { out.append((char*)txt, len); lexbor_free(txt); }
        return;
    }
    bool isBlock = false;
    for (int i = 0; BLOCK[i]; ++i)
        if (tagEq(node, BLOCK[i])) { isBlock = true; break; }

    if (isBlock && !out.empty() && out.back() != '\n')
        out += '\n';

    lxb_dom_node_t* child = lxb_dom_node_first_child(node);
    while (child) { nodeToText(child, out); child = lxb_dom_node_next(child); }
}

// ---- 公開関数 ----

std::string extractContent(const std::string& html) {
    lxb_html_document_t* doc = lxb_html_document_create();
    if (!doc) return stripHtml(html);

    lxb_status_t st = lxb_html_document_parse(
        doc, (const lxb_char_t*)html.c_str(), html.size());
    if (st != LXB_STATUS_OK) {
        lxb_html_document_destroy(doc);
        return stripHtml(html);
    }

    lxb_dom_node_t* body = lxb_dom_interface_node(
        lxb_html_document_body_element(doc));
    if (!body) {
        lxb_html_document_destroy(doc);
        return stripHtml(html);
    }

    // 1. ノイズ除去
    removeNoiseNodes(body);

    // 2. article/main が存在すれば即採用
    lxb_dom_node_t* winner = findPriorityNode(body);

    // 3. article/main がなければ div/section をスコアリング
    if (!winner) {
        double bestScore = 0.0;
        lxb_dom_node_t* node = lxb_dom_node_first_child(body);
        while (node) {
            for (int i = 0; CANDIDATE_TAGS[i]; ++i) {
                if (tagEq(node, CANDIDATE_TAGS[i])) {
                    size_t totalLen;
                    lxb_char_t* txt = lxb_dom_node_text_content(node, &totalLen);
                    if (txt) {
                        lexbor_free(txt);
                        double lk = (totalLen > 0)
                            ? (double)linkTextLen(node) / totalLen : 0.0;
                        double s = scoreNode(node) * (1.0 - lk);
                        if (s > bestScore) { bestScore = s; winner = node; }
                    }
                }
            }
            node = lxb_dom_node_next(node);
        }
    }

    // 4. 勝者なしは body 全体をフォールバック
    if (!winner) winner = body;

    // 5. テキスト化
    std::string result;
    result.reserve(4096);
    nodeToText(winner, result);

    lxb_html_document_destroy(doc);

    // 空なら従来の stripHtml にフォールバック
    if (result.empty()) return stripHtml(html);

    // 連続空行を正規化（stripHtml と同じ処理）
    std::string normalized;
    normalized.reserve(result.size());
    int nl = 0;
    for (char c : result) {
        if (c == '\n') { if (++nl <= 2) normalized += c; }
        else           { nl = 0; normalized += c; }
    }
    return normalized;
}
```

## 4. 各機能の詳細設計

### FR-1: 本文抽出精度の向上

`extractContent()` の処理フロー:

```
parse HTML → removeNoiseNodes(body)
    │
    ├─ findPriorityNode() で article/main 発見 → 即採用
    │
    └─ 未発見 → body の直下 div/section を走査
           各ノード: score = scoreNode() × (1 - linkDensity())
           最高スコアのノード → winner
               │
               └─ score=0 or なし → body 全体
    │
    └─ nodeToText(winner) → 正規化 → return
```

スコアが 0.0 だった場合（本文段落が一切ない）は body 全体 → テキスト化 → 空なら `stripHtml()` フォールバック。

### FR-2: HN型フィード対応

`ui.cpp` の既存実装がそのまま機能する（変更不要）:
- `art.content.size() < 200` ならリンクをフェッチ
- フェッチ失敗時は既存の短い説明テキストを維持（上書きしない）
- `fetchArticleBody()` が Lexbor ベースに切り替わることで抽出品質が向上

### FR-3: Lexbor 導入

- アマルガメーション生成はホスト（Windows/MSYS2）で実施。3DS ビルドには影響しない
- `lexbor.c` は C99 コード。`CFLAGS` はそのまま通用するはず（ARM実績あり）
- 万一コンパイルエラーが出た場合は gumbo-parser（移管後系）に切り替えて同じ `extractContent()` インタフェースを実装する（RDD 代替案）
