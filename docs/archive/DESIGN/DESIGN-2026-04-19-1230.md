# システム設計書 - Phase 7 画像表示 MVP
**RDD:** `RDD-2026-04-19-1200.md`
**作成日:** 2026-04-19

## 1. 全体方針

- **画像デコーダ**: `lib/stb/stb_image.h`（header-only、devkitPro の Switch サンプルに同梱されているものを流用）
- **ダウンロード**: 既存の `net.cpp` に **バイナリ取得関数** を追加し、Content-Length と受信バイト数で2MBキャップを強制
- **URL 抽出**: `readability.cpp` のツリー走査中に `<img>` を検出して `src` を収集する
- **画像パイプライン**: 新規 `source/image_loader.cpp/.h` に「URL→デコード済みRGBA→3DSテクスチャ」のステートマシンを置く
- **バックグラウンド取得**: libctru の `Thread` + `LightLock` によるワーカースレッド1本。本文描画は即座に始まり、画像は完成次第差し替え
- **描画**: `ArticleView` の本文行列末尾に画像セクションを描画。スクロール位置は「行数×LINE_HEIGHT + 画像合計高さ」で統一管理
- **テクスチャライフサイクル**: 記事遷移時に worker を joinしてから全テクスチャ解放。これにより GPU コマンド実行中の delete を避ける

## 2. 新規ファイル

### 2.1 `lib/stb/stb_image.h`

### 2.2 `source/image_loader.h`

```cpp
#pragma once
#include <string>
#include <vector>
#include <citro2d.h>
#include <3ds.h>

enum class ImageStatus : u8 {
    Pending,   // キューに入っているが未着手
    Loading,   // ダウンロード/デコード中
    Ready,     // テクスチャ化完了
    Failed,    // 失敗（プレースホルダ表示）
};

struct LoadedImage {
    std::string url;
    ImageStatus status = ImageStatus::Pending;

    // Ready の時のみ有効
    C3D_Tex    tex{};             // VRAM テクスチャ（POT パディング済み）
    Tex3DS_SubTexture sub{};      // 有効領域（元画像のアスペクトそのまま）
    int        displayW = 0;      // 実描画幅（縮小後）
    int        displayH = 0;
};

// 記事単位のローダ。画像URLを受け取り、裏でダウンロード→デコード→テクスチャ化する。
// 生成後に start() を呼び、ArticleView 終了時に stop() を呼ぶこと。
class ImageLoader {
public:
    ImageLoader();
    ~ImageLoader();

    // URL配列を元にロードキューを初期化。既にstart済みならstopしてから呼ぶこと。
    void reset(const std::vector<std::string>& urls);

    // ワーカースレッド起動。すでに走っていれば何もしない。
    void start();

    // ワーカー停止 & 全テクスチャ解放（C3D_TexDelete）。スレッドjoinまでブロック。
    // GPU描画フレーム外（frameBegin/frameEnd の外）で呼ぶこと。
    void stop();

    // 現在の状態スナップショット。描画スレッドから毎フレーム呼ばれる想定。
    // LightLock を取ってコピーを返す。
    std::vector<LoadedImage> snapshot() const;

    // 進捗（Ready + Failed 件数 / 総数）
    void progress(int& doneOut, int& totalOut) const;

private:
    void workerMain();
    static void workerTrampoline(void* self);

    mutable LightLock      lock_{};
    std::vector<LoadedImage> items_;
    Thread                 thread_ = nullptr;
    volatile bool          stopRequested_ = false;
    volatile bool          running_ = false;
};
```

### 2.3 `source/image_loader.cpp`

```cpp
#define STB_IMAGE_IMPLEMENTATION
#define STBI_ONLY_JPEG
#define STBI_ONLY_PNG
#define STBI_NO_STDIO
#include "stb_image.h"

#include "image_loader.h"
#include "net.h"
#include <cstring>
#include <algorithm>

static constexpr size_t MAX_IMAGE_BYTES = 2 * 1024 * 1024;   // 2MB
static constexpr int    TARGET_WIDTH    = 400;               // 上画面幅

// POT丸め
static int nextPOT(int v) { int r = 8; while (r < v) r <<= 1; return r; }

// --- 静的ヘルパ: RGBA ピクセルを 3DS テクスチャのタイル順に swizzle ---
// citro2d の C2D_Image が期待する順序。
// 参考: libctru Tex3DS のタイル配置。
static void swizzleRGBA8ToTex(const uint8_t* src, int srcW, int srcH,
                              uint8_t* dst, int texW, int texH) {
    // 8x8 Morton タイル順
    for (int ty = 0; ty < texH; ty += 8) {
      for (int tx = 0; tx < texW; tx += 8) {
        for (int py = 0; py < 8; ++py) {
          for (int px = 0; px < 8; ++px) {
            int x = tx + px;
            int y = ty + py;
            // Z-order within tile: see libctru (x,y)→Morton
            uint32_t morton =
              ((x & 1) << 0) | ((y & 1) << 1) |
              ((x & 2) << 1) | ((y & 2) << 2) |
              ((x & 4) << 2) | ((y & 4) << 3);
            int dstIdx = (ty / 8) * (texW / 8) * 64 + (tx / 8) * 64 + morton;
            uint8_t* d = dst + dstIdx * 4;
            if (x < srcW && y < srcH) {
              const uint8_t* s = src + (y * srcW + x) * 4;
              // 3DS テクスチャは ABGR バイト順
              d[0] = s[3]; d[1] = s[2]; d[2] = s[1]; d[3] = s[0];
            } else {
              d[0] = 0; d[1] = 0; d[2] = 0; d[3] = 0;
            }
          }
        }
      }
    }
}

// --- 静的ヘルパ: 画像を TARGET_WIDTH に収めるための縮小倍率を決める ---
static float computeScale(int w) {
    if (w <= TARGET_WIDTH) return 1.0f;
    return (float)TARGET_WIDTH / (float)w;
}

// --- 静的ヘルパ: 最近傍で縮小 ---
static void nearestResize(const uint8_t* src, int sw, int sh,
                          uint8_t* dst, int dw, int dh) {
    for (int y = 0; y < dh; ++y) {
        int sy = y * sh / dh;
        for (int x = 0; x < dw; ++x) {
            int sx = x * sw / dw;
            memcpy(dst + (y * dw + x) * 4, src + (sy * sw + sx) * 4, 4);
        }
    }
}

ImageLoader::ImageLoader() { LightLock_Init(&lock_); }
ImageLoader::~ImageLoader() { stop(); }

void ImageLoader::reset(const std::vector<std::string>& urls) {
    stop();
    LightLock_Lock(&lock_);
    items_.clear();
    items_.reserve(urls.size());
    for (const auto& u : urls) {
        LoadedImage it;
        it.url = u;
        it.status = ImageStatus::Pending;
        items_.push_back(std::move(it));
    }
    LightLock_Unlock(&lock_);
}

void ImageLoader::start() {
    if (running_) return;
    stopRequested_ = false;
    running_ = true;
    thread_ = threadCreate(workerTrampoline, this, 32 * 1024, 0x3F, -2, false);
}

void ImageLoader::stop() {
    if (!running_) return;
    stopRequested_ = true;
    if (thread_) {
        threadJoin(thread_, U64_MAX);
        threadFree(thread_);
        thread_ = nullptr;
    }
    running_ = false;

    // 全テクスチャ解放（join 後なので安全）
    LightLock_Lock(&lock_);
    for (auto& it : items_) {
        if (it.status == ImageStatus::Ready) {
            C3D_TexDelete(&it.tex);
        }
    }
    items_.clear();
    LightLock_Unlock(&lock_);
}

std::vector<LoadedImage> ImageLoader::snapshot() const {
    LightLock_Lock(&lock_);
    auto copy = items_;  // C3D_Tex の値コピーは参照だけ（テクスチャメモリは共有）
    LightLock_Unlock(&lock_);
    return copy;
}

void ImageLoader::progress(int& doneOut, int& totalOut) const {
    LightLock_Lock(&lock_);
    int done = 0;
    for (auto& it : items_) {
        if (it.status == ImageStatus::Ready || it.status == ImageStatus::Failed) ++done;
    }
    doneOut = done; totalOut = (int)items_.size();
    LightLock_Unlock(&lock_);
}

void ImageLoader::workerTrampoline(void* self) {
    static_cast<ImageLoader*>(self)->workerMain();
}

void ImageLoader::workerMain() {
    for (size_t i = 0; ; ++i) {
        if (stopRequested_) return;

        std::string url;
        LightLock_Lock(&lock_);
        if (i >= items_.size()) { LightLock_Unlock(&lock_); return; }
        url = items_[i].url;
        items_[i].status = ImageStatus::Loading;
        LightLock_Unlock(&lock_);

        // 1. ダウンロード（サイズキャップ付き）
        std::string errMsg;
        std::vector<uint8_t> buf = httpGetBinary(url, MAX_IMAGE_BYTES, errMsg);
        if (buf.empty() || stopRequested_) {
            LightLock_Lock(&lock_);
            if (i < items_.size()) items_[i].status = ImageStatus::Failed;
            LightLock_Unlock(&lock_);
            continue;
        }

        // 2. デコード
        int w = 0, h = 0, ch = 0;
        uint8_t* pixels = stbi_load_from_memory(buf.data(), (int)buf.size(),
                                                 &w, &h, &ch, 4);
        if (!pixels) {
            LightLock_Lock(&lock_);
            if (i < items_.size()) items_[i].status = ImageStatus::Failed;
            LightLock_Unlock(&lock_);
            continue;
        }

        // 3. 縮小
        float sc = computeScale(w);
        int dw = (int)(w * sc), dh = (int)(h * sc);
        std::vector<uint8_t> scaled;
        const uint8_t* finalPixels;
        if (sc < 1.0f) {
            scaled.resize(dw * dh * 4);
            nearestResize(pixels, w, h, scaled.data(), dw, dh);
            finalPixels = scaled.data();
        } else {
            finalPixels = pixels;
            dw = w; dh = h;
        }

        // 4. POT に切り上げてテクスチャ確保
        int texW = nextPOT(dw), texH = nextPOT(dh);
        if (texW > 1024 || texH > 1024) {
            stbi_image_free(pixels);
            LightLock_Lock(&lock_);
            if (i < items_.size()) items_[i].status = ImageStatus::Failed;
            LightLock_Unlock(&lock_);
            continue;
        }

        C3D_Tex tex;
        if (!C3D_TexInit(&tex, texW, texH, GPU_RGBA8)) {
            stbi_image_free(pixels);
            LightLock_Lock(&lock_);
            if (i < items_.size()) items_[i].status = ImageStatus::Failed;
            LightLock_Unlock(&lock_);
            continue;
        }

        // 5. swizzle してアップロード
        swizzleRGBA8ToTex(finalPixels, dw, dh,
                          (uint8_t*)tex.data, texW, texH);

        stbi_image_free(pixels);

        // 6. 結果をコミット
        LightLock_Lock(&lock_);
        if (i < items_.size()) {
            items_[i].tex       = tex;
            items_[i].sub.width  = (u16)dw;
            items_[i].sub.height = (u16)dh;
            items_[i].sub.left   = 0.0f;
            items_[i].sub.top    = 1.0f;
            items_[i].sub.right  = (float)dw / (float)texW;
            items_[i].sub.bottom = 1.0f - (float)dh / (float)texH;
            items_[i].displayW   = dw;
            items_[i].displayH   = dh;
            items_[i].status     = ImageStatus::Ready;
        } else {
            C3D_TexDelete(&tex);
        }
        LightLock_Unlock(&lock_);
    }
}
```

### 3.2 `source/net.cpp` — `httpGetBinary` 実装

```cpp
struct BinaryCtx {
    std::vector<uint8_t>* buf;
    size_t               maxBytes;
    bool                 aborted;
};

static size_t writeBinaryCallback(char* ptr, size_t sz, size_t nm, void* ud) {
    auto* ctx = static_cast<BinaryCtx*>(ud);
    size_t n = sz * nm;
    if (ctx->aborted) return 0;
    if (ctx->buf->size() + n > ctx->maxBytes) {
        ctx->aborted = true;
        return 0; // libcurl に中断を伝える
    }
    ctx->buf->insert(ctx->buf->end(), ptr, ptr + n);
    return n;
}

std::vector<uint8_t> httpGetBinary(const std::string& url,
                                    size_t maxBytes,
                                    std::string& errMsg) {
    CURL* curl = curl_easy_init();
    if (!curl) { errMsg = "curl_easy_init failed"; return {}; }

    std::vector<uint8_t> buf;
    BinaryCtx ctx { &buf, maxBytes, false };

    curl_easy_setopt(curl, CURLOPT_URL,             url.c_str());
    curl_easy_setopt(curl, CURLOPT_WRITEFUNCTION,   writeBinaryCallback);
    curl_easy_setopt(curl, CURLOPT_WRITEDATA,       &ctx);
    curl_easy_setopt(curl, CURLOPT_FOLLOWLOCATION,  1L);
    curl_easy_setopt(curl, CURLOPT_TIMEOUT,         30L);
    curl_easy_setopt(curl, CURLOPT_SSL_VERIFYPEER,  1L);
    curl_easy_setopt(curl, CURLOPT_SSL_VERIFYHOST,  2L);
    curl_easy_setopt(curl, CURLOPT_CAINFO,          "sdmc:/3ds/rssreader/cacert.pem");
    curl_easy_setopt(curl, CURLOPT_USERAGENT,       "3DS-RSSReader/1.0");
    curl_easy_setopt(curl, CURLOPT_MAXFILESIZE_LARGE, (curl_off_t)maxBytes);

    CURLcode res = curl_easy_perform(curl);
    curl_easy_cleanup(curl);

    if (ctx.aborted) { errMsg = "size limit exceeded"; return {}; }
    if (res != CURLE_OK) { errMsg = curl_easy_strerror(res); return {}; }
    return buf;
}
```

### 3.7 `source/ui.cpp` — `drawArticleView` 拡張

変更点のみ（既存テキスト描画は維持）:

1. **記事切替検出**: `cachedImagesFeed/Article != selected` なら：
   - `state.imgLoader.reset(art.imageUrls)` → `start()`
   - `cachedImagesFeed/Article` を更新

2. **画面離脱検出**: `uiHandleInput` で B 押下→`ArticleList` 遷移時、または記事変更時に `state.imgLoader.stop()`

3. **画像セクション描画**: 本文行列末尾の Y 位置から：
   ```
   ── 画像 ──
   [取得中 2/5]       ← snapshot() 由来の進捗
   画像1（Ready ならテクスチャ描画、Failed なら "[画像取得失敗]"）
   画像2
   ...
   ```
   - 画像高さ + 6px マージンを積む
   - スクロール可能範囲に加算し、`maxScroll` を再計算

4. **スクロール最大値**:
   ```cpp
   int textPixels = totalLines * LINE_HEIGHT;
   int imgPixels = <snapshotから合計>;
   int totalPixels = textPixels + imgPixels;
   int maxScroll_px = max(0, totalPixels - visibleHeight);
   ```
   既存は行単位スクロール。画像混在後はピクセル単位に統一する。

   ※ スコープ影響を抑えるため、MVP では「画像を描画するが、画像はページの最下段にあって見るには最後までスクロール要」とし、スクロール単位は既存の行単位を維持する簡易版でもよい。**推奨: 行単位スクロールのまま、最下段まで達した後さらに下にスクロール可能とし、画像は画面下から追加で表示される** ようにする。詳細は実装時に実機で調整。

## 4. 各機能の詳細設計

### FR-1 画像URL抽出
- `readability.cpp` の本文判定後の subtree walk 内で `tagEq(n, "img")` を検出
- `getAttr(n, "src", &len)` → `std::string`
- 相対URL解決：記事URLをパースして `scheme://host` / base path を取り出すユーティリティ（新規 `static std::string resolveUrl(const std::string& base, const std::string& ref)`）
- 最大10件を超えたら収集停止

### FR-2〜5 ダウンロード→デコード→テクスチャ
- ワーカースレッドで逐次処理（並列化しない：メモリ節約）
- 流れは `ImageLoader::workerMain` を参照

### FR-6 バックグラウンド取得 + 即時本文表示
- `drawArticleView` 初回呼出時に `reset + start`
- 本文描画はスレッド完了を待たない
- 毎フレーム `snapshot()` を取り、Ready の画像を描画する

### FR-7 プレースホルダ
- `Failed` 状態の画像は「[画像取得失敗]」テキスト1行を画像の位置に描画

### FR-8 最大10枚
- `readability.cpp` の収集段階で打ち切り

### FR-9 2MB 上限
- `httpGetBinary` の `CURLOPT_MAXFILESIZE_LARGE` と writeBinaryCallback 内の再チェックで二重防御

## 7. リスクと対策

| リスク | 対策 |
|---|---|
| テクスチャ swizzle コードの誤り → 画像が乱れる | GPU_RGBA8 に代えて `C3D_SyncTextureCopy` で DMA コピーも検討。実装時に libctru サンプル参照 |
| Old 3DS でメモリ不足 | 10枚×512×256 RGBA ≒ 5MB VRAM。VRAM 6MB 制約に収まるが余裕少 → 実機で確認し、超過時は最大枚数を下げる |
| threadCreate の stack 32KB が不足（libcurl+stb_image 展開） | 64KB へ引き上げる余地あり。実装時にメモリダンプで確認 |
| `C3D_TexInit` を worker スレッドから呼ぶ安全性 | libctru のテクスチャAPI は main thread 前提の箇所あり。**重要**：実装前に libctru サンプル/Issue で確認。問題あれば「worker はデコード/縮小まで、main thread で C3D_TexInit + swizzle」の二段構えに変更する |
