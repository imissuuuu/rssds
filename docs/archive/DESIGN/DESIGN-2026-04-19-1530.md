# システム設計書 - Phase 7 Stage 2 実画像描画
**RDD:** `RDD-2026-04-19-1500.md`
**作成日:** 2026-04-19

## 1. 全体方針

### スレッド分担（PDF リサーチ結論に準拠）
- **Worker thread (1本)**: download → decode (stb_image) → resize。出力は **heap 上の RGBA8 配列**（`std::vector<uint8_t>`）+ サイズメタ
- **Render thread (= main thread)**: pop result → `C3D_TexInit` → swizzle を直接 `tex.data` へ書き込み (A,B,G,R + 8×8 Morton) → `C3D_TexFlush` → free pixel buf → cache 登録
- **理由**: libctru allocator (`linearAlloc`) と citro3d API には内部同期がないため、worker からは一切呼ばない。worker は heap (`malloc`/`std::vector`) のみ。render thread が allocator/GPU を独占する

### Allocator 戦略
- テクスチャは `linearAlloc`（`onVram=false`）標準。VRAM bank 偏り問題を完全回避
- `C3D_TexInit(format=GPU_RGBA8)` がそのまま linearAlloc を経由

### キャッシュ戦略
- `ImageCache` は url → `CachedImage` の map。可視範囲±1枚を keep
- 可視判定: スクロール Y と各画像の Y 位置 (本文末尾から累積) から算出
- 範囲外は `C3D_TexDelete` で即解放

### レートリミット
- Render thread での `C3D_TexInit + swizzle + TexFlush` は 1 フレーム最大 1 枚（NFR-5）
- swizzle は 256×256 で約 256 KiB の memcpy 相当 → 1 フレームで安全

### 通信
- Worker ⇄ Render は `LightLock` で保護した job queue + result queue。`std::atomic` 禁止（coding-patterns rule #5）

### 2.2 `source/image_loader.h`

```cpp
#pragma once
#include <3ds.h>
#include <string>
#include <vector>
#include <deque>
#include <cstdint>

// worker thread が生成し、render thread が消費するデコード済み画像
struct DecodedImage {
    std::string url;
    std::vector<uint8_t> rgba;  // imgW*imgH*4 byte, top-left 起点
    int imgW = 0;
    int imgH = 0;
    bool failed = false;        // true なら rgba 空、render 側はプレースホルダ表示
};

// download/decode/resize を担う worker。
// 起動: start(), 停止: stop() (join まで block)。
// submit() で URL を投入、poll() で完了結果を取り出す。
class ImageLoader {
public:
    ImageLoader();
    ~ImageLoader();

    void start();
    void stop();

    // 未着手 URL を投入。重複投入防止は呼び出し側責任 (cache が in-flight set を持つ)
    void submit(const std::string& url);

    // 全未処理 job を破棄 (記事遷移時)。in-flight 1件は完走を待つ
    void cancelAll();

    // 完了結果を 1 件取得。なければ false
    bool poll(DecodedImage& out);

private:
    void workerMain();
    static void trampoline(void* self);

    LightLock              lock_;
    std::deque<std::string> jobs_;
    std::deque<DecodedImage> results_;
    Thread                 thread_  = nullptr;
    volatile bool          running_ = false;
    volatile bool          stop_    = false;
    LightEvent             wakeup_;  // job 追加時に worker を起こす
};
```

### 2.3 `source/image_loader.cpp`（要点のみ）

```cpp
#define STB_IMAGE_IMPLEMENTATION
#define STBI_ONLY_JPEG
#define STBI_ONLY_PNG
#define STBI_NO_STDIO
#include "stb_image.h"
#include "image_loader.h"
#include "net.h"

static constexpr int    MAX_DIM       = 256;       // 縮小先上限 (256×256 POT)
static constexpr size_t MAX_BYTES     = 2 * 1024 * 1024;
static constexpr size_t WORKER_STACK  = 64 * 1024;

// バイリニア縮小: 出力 dw×dh は MAX_DIM 以下、アスペクト比維持
static void bilinearResize(const uint8_t* src, int sw, int sh,
                            uint8_t* dst, int dw, int dh);

ImageLoader::ImageLoader() {
    LightLock_Init(&lock_);
    LightEvent_Init(&wakeup_, RESET_STICKY);
}
ImageLoader::~ImageLoader() { stop(); }

void ImageLoader::start() {
    if (running_) return;
    stop_ = false; running_ = true;
    thread_ = threadCreate(trampoline, this, WORKER_STACK, 0x3F, -2, false);
}

void ImageLoader::stop() {
    if (!running_) return;
    stop_ = true;
    LightEvent_Signal(&wakeup_);
    threadJoin(thread_, U64_MAX);
    threadFree(thread_);
    thread_ = nullptr;
    running_ = false;
    LightLock_Lock(&lock_);
    jobs_.clear();
    results_.clear();
    LightLock_Unlock(&lock_);
}

void ImageLoader::submit(const std::string& url) {
    LightLock_Lock(&lock_);
    jobs_.push_back(url);
    LightLock_Unlock(&lock_);
    LightEvent_Signal(&wakeup_);
}

void ImageLoader::cancelAll() {
    LightLock_Lock(&lock_);
    jobs_.clear();
    LightLock_Unlock(&lock_);
}

bool ImageLoader::poll(DecodedImage& out) {
    LightLock_Lock(&lock_);
    if (results_.empty()) { LightLock_Unlock(&lock_); return false; }
    out = std::move(results_.front());
    results_.pop_front();
    LightLock_Unlock(&lock_);
    return true;
}

void ImageLoader::trampoline(void* self) {
    static_cast<ImageLoader*>(self)->workerMain();
}

void ImageLoader::workerMain() {
    while (!stop_) {
        std::string url;
        LightLock_Lock(&lock_);
        if (jobs_.empty()) {
            LightLock_Unlock(&lock_);
            LightEvent_Wait(&wakeup_);
            LightEvent_Clear(&wakeup_);
            continue;
        }
        url = std::move(jobs_.front());
        jobs_.pop_front();
        LightLock_Unlock(&lock_);

        DecodedImage out;
        out.url = url;

        std::string err;
        std::vector<uint8_t> bin = httpGetBinary(url, MAX_BYTES, err);
        if (bin.empty()) { out.failed = true; goto push; }

        int w, h, ch;
        if (uint8_t* px = stbi_load_from_memory(bin.data(), (int)bin.size(),
                                                 &w, &h, &ch, 4)) {
            // 長辺を MAX_DIM に合わせ縮小 (アスペクト比維持)
            int dw, dh;
            if (w >= h) { dw = std::min(w, MAX_DIM); dh = h * dw / w; }
            else        { dh = std::min(h, MAX_DIM); dw = w * dh / h; }
            if (dw < 1) dw = 1;
            if (dh < 1) dh = 1;

            out.imgW = dw; out.imgH = dh;
            out.rgba.resize(dw * dh * 4);
            if (dw == w && dh == h) memcpy(out.rgba.data(), px, dw*dh*4);
            else                    bilinearResize(px, w, h, out.rgba.data(), dw, dh);

            stbi_image_free(px);
        } else {
            out.failed = true;
        }
    push:
        LightLock_Lock(&lock_);
        results_.push_back(std::move(out));
        LightLock_Unlock(&lock_);
    }
}
```

### 2.4 `source/image_cache.h`

```cpp
#pragma once
#include <citro2d.h>
#include <3ds.h>
#include <string>
#include <vector>
#include <unordered_map>
#include <unordered_set>
#include "image_loader.h"

enum class ImgState : uint8_t { None, Pending, Ready, Failed };

struct CachedImage {
    ImgState state = ImgState::None;
    C3D_Tex   tex{};
    Tex3DS_SubTexture sub{};
    bool      texInited = false;
    int       imgW = 0;   // 描画サイズ
    int       imgH = 0;
};

// render thread からのみ操作。worker (ImageLoader) と疎結合。
class ImageCache {
public:
    void attach(ImageLoader* loader) { loader_ = loader; }

    // 記事遷移時。全 tex を解放し loader に cancelAll を伝える
    void resetForArticle(const std::vector<std::string>& urls);

    // 毎フレーム呼ぶ:
    //   1. visibleUrls にあって state==None のものを loader->submit + Pending化
    //   2. visibleUrls にないものは TexDelete + 削除
    //   3. loader->poll() を最大 1 件処理し TexInit+swizzle+Flush
    void tick(const std::unordered_set<std::string>& visibleUrls);

    // 描画用に取得 (順序は urls_ 配列順)
    const CachedImage* get(const std::string& url) const;

private:
    void uploadOne(DecodedImage&& d);
    void evict(const std::string& url);

    ImageLoader* loader_ = nullptr;
    std::vector<std::string> urls_;  // 記事の画像順 (描画順序と Y 位置算出に使用)
    std::unordered_map<std::string, CachedImage> map_;
};
```

### 2.5 `source/image_cache.cpp`（要点のみ）

```cpp
#include "image_cache.h"
#include <cstring>

static int nextPOT(int v) { int r = 8; while (r < v) r <<= 1; return r; }

// (x,y) (0..7) → 0..63 Morton index
static inline uint32_t morton8(int x, int y) {
    return ((x & 1) << 0) | ((y & 1) << 1)
         | ((x & 2) << 1) | ((y & 2) << 2)
         | ((x & 4) << 2) | ((y & 4) << 3);
}

// PDF: A,B,G,R byte 順 + 8×8 タイル Morton swizzle
// dst は texW*texH*4 byte 既確保。imgW/imgH はパディング前の有効領域。
static void swizzleABGR(const uint8_t* src, int imgW, int imgH,
                        uint8_t* dst, int texW, int texH) {
    for (int y = 0; y < texH; ++y) {
        int ty = y >> 3, py = y & 7;
        for (int x = 0; x < texW; ++x) {
            int tx = x >> 3, px = x & 7;
            int dstIdx = (ty * (texW >> 3) + tx) * 64 + morton8(px, py);
            uint8_t* d = dst + dstIdx * 4;
            if (x < imgW && y < imgH) {
                const uint8_t* s = src + (y * imgW + x) * 4;
                d[0] = s[3]; d[1] = s[2]; d[2] = s[1]; d[3] = s[0];  // A,B,G,R
            } else {
                d[0] = d[1] = d[2] = d[3] = 0;
            }
        }
    }
}

void ImageCache::resetForArticle(const std::vector<std::string>& urls) {
    if (loader_) loader_->cancelAll();
    // 完了済み in-flight を吸収 (drain 1 round)
    DecodedImage d;
    while (loader_ && loader_->poll(d)) { /* discard */ }
    for (auto& kv : map_) {
        if (kv.second.texInited) C3D_TexDelete(&kv.second.tex);
    }
    map_.clear();
    urls_ = urls;
    for (const auto& u : urls_) map_[u] = {};
}

void ImageCache::tick(const std::unordered_set<std::string>& vis) {
    // 1. 不要なものを evict
    for (auto it = map_.begin(); it != map_.end(); ++it) {
        if (vis.count(it->first)) continue;
        if (it->second.texInited) {
            C3D_TexDelete(&it->second.tex);
            it->second.texInited = false;
        }
        it->second.state = ImgState::None;
    }
    // 2. 必要なものを submit (state==None のみ)
    for (const auto& u : vis) {
        auto& c = map_[u];
        if (c.state == ImgState::None && loader_) {
            loader_->submit(u);
            c.state = ImgState::Pending;
        }
    }
    // 3. 完了を 1 件 upload (NFR-5: 1 フレーム 1 枚)
    DecodedImage d;
    if (loader_ && loader_->poll(d)) {
        // 既に evict 済み or 別記事の残骸なら破棄
        auto it = map_.find(d.url);
        if (it == map_.end()) return;
        if (!vis.count(d.url)) { it->second.state = ImgState::None; return; }
        uploadOne(std::move(d));
    }
}

void ImageCache::uploadOne(DecodedImage&& d) {
    auto& c = map_[d.url];
    if (d.failed) { c.state = ImgState::Failed; return; }

    int texW = nextPOT(d.imgW), texH = nextPOT(d.imgH);
    if (texW > 256 || texH > 256) { c.state = ImgState::Failed; return; }

    if (!C3D_TexInit(&c.tex, texW, texH, GPU_RGBA8)) {
        c.state = ImgState::Failed; return;
    }
    c.texInited = true;
    swizzleABGR(d.rgba.data(), d.imgW, d.imgH,
                (uint8_t*)c.tex.data, texW, texH);
    C3D_TexFlush(&c.tex);

    c.imgW = d.imgW; c.imgH = d.imgH;
    c.sub.width  = (u16)d.imgW;
    c.sub.height = (u16)d.imgH;
    c.sub.left   = 0.0f;
    c.sub.right  = (float)d.imgW / (float)texW;
    c.sub.top    = 1.0f;
    c.sub.bottom = 1.0f - (float)d.imgH / (float)texH;
    c.state = ImgState::Ready;
}

const CachedImage* ImageCache::get(const std::string& url) const {
    auto it = map_.find(url);
    return it == map_.end() ? nullptr : &it->second;
}
```

## 7. リスクと対策

| リスク | 対策 |
|---|---|
| Worker thread の stack 64 KiB 不足 (libcurl + stb_image 同時展開) | 不足を観測したら 128 KiB へ増設。実機ログで stack 残量確認手段なし → 落ちたら対応 |
| Swizzle 計算ミスで画像が乱れる | 8×8 単位で raw メモリダンプ可能。tinyxml2 sample (256×256 単色) を投入して目視 |
| `linearAlloc` の同時呼び出し競合 | render thread のみが触る設計のため発生しない（worker は heap のみ） |
| メモリリーク（記事連続切替） | `resetForArticle` で全 `TexDelete`、`stop()` でも保険的に解放 |
| 記事スクロール時に大量 evict/submit が往復 | 可視判定に ±1 枚ヒステリシスを入れる（ある程度離れるまで keep） |
| `ImgState::Failed` の URL を再 submit してしまう | tick の "state==None のみ submit" 条件で防ぐ。Failed は state を維持 |
