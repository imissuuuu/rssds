**RDD:** `RDD-2026-04-24-1400.md`
**作成日:** 2026-04-24

## 1. 全体方針

- **CI は 3 ジョブ構成**: `format`（clang-format）、`build`（devkitpro/devkitarm + cppcheck）、`clang-tidy`（試行・non-blocking）
- **cppcheck は build ジョブ内で実行**: devkitpro コンテナ内に devkitPro ヘッダがあるため、`-I` パスが正確に通る
- **clang-tidy は `continue-on-error: true` で試行**: ARM フラグ問題の解決度に応じて後から必須化する
- **`-Werror` は GitHub Actions の環境変数 `CI=true` を活用**: Makefile に `ifdef CI` ブロックを追加するだけで CI と手元を分離できる
- **commit-dev.sh の clang-format チェックは non-fatal**: clang-format が PATH にない環境（Windows で未インストール）はスキップして警告のみ
- **DevkitPro Docker タグは `20260221` で固定**（NFR-1）

### `.github/workflows/ci.yml`

```yaml
name: CI

on:
  pull_request:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read

jobs:
  # ── ゲート 1: フォーマットチェック ──────────────────────────────
  format:
    name: clang-format
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install clang-format
        run: sudo apt-get update && sudo apt-get install -y clang-format

      - name: Check formatting
        run: |
          files=$(git ls-files '*.cpp' '*.c' '*.h' '*.hpp')
          if [ -n "$files" ]; then
            echo "$files" | xargs clang-format --dry-run --Werror -style=file
          fi

  # ── ゲート 2: ビルド確認 + cppcheck ────────────────────────────
  build:
    name: Build & Static Analysis
    runs-on: ubuntu-latest
    container:
      image: devkitpro/devkitarm:20260221
      options: --user 1001
    steps:
      - uses: actions/checkout@v4
        with:
          submodules: recursive
          fetch-depth: 0

      - name: Install cppcheck
        run: apt-get update && apt-get install -y cppcheck

      - name: Build
        run: make -j$(nproc)

      - name: Verify output
        run: test -f rssds.3dsx

      - name: Upload artifacts
        uses: actions/upload-artifact@v4
        with:
          name: rssds-build
          path: |
            rssds.3dsx
            rssds.elf

      - name: cppcheck
        run: |
          cppcheck \
            --enable=all \
            --language=c++ \
            --platform=unix32 \
            --check-level=normal \
            --suppress=missingIncludeSystem \
            --suppress=unmatchedSuppression \
            -I /opt/devkitpro/libctru/include \
            -I /opt/devkitpro/portlibs/3ds/include \
            --error-exitcode=1 \
            source/

  # ── 試行: clang-tidy（non-blocking）────────────────────────────
  clang-tidy:
    name: clang-tidy (experimental)
    runs-on: ubuntu-latest
    continue-on-error: true          # 失敗してもマージをブロックしない
    container:
      image: devkitpro/devkitarm:20260221
      options: --user 1001
    steps:
      - uses: actions/checkout@v4
        with:
          submodules: recursive
          fetch-depth: 0

      - name: Install tools
        run: apt-get update && apt-get install -y bear clang-tidy python3

      - name: Generate compile_commands.json
        run: bear -- make -j$(nproc)

      - name: Strip ARM-specific flags
        run: python3 .claude/scripts/strip_arm_flags.py

      - name: Run clang-tidy
        run: |
          run-clang-tidy \
            -p . \
            -header-filter='(source|include)/.*' \
            -extra-arg='-std=gnu++17' \
            -extra-arg='--query-driver=/opt/devkitpro/devkitARM/bin/arm-none-eabi-g++' \
            $(find source -name '*.cpp')
```

### `.claude/scripts/strip_arm_flags.py`

ARM 固有フラグを compile_commands.json から除去するスクリプト（clang-tidy に渡す前に実行）。

```python
#!/usr/bin/env python3
"""Remove ARM/devkitPro-specific compiler flags that clang-tidy doesn't understand."""
import json, shlex

REMOVE_FLAGS = {
    '-mword-relocations', '-mfloat-abi=hard', '-mfloat-abi=softfp',
    '-mtp=soft', '-fno-rtti', '-fno-exceptions',
    '-fomit-frame-pointer', '-ffunction-sections',
}
REMOVE_PREFIXES = ('-specs=', '-march=', '-mtune=')

def should_remove(flag: str) -> bool:
    return flag in REMOVE_FLAGS or any(flag.startswith(p) for p in REMOVE_PREFIXES)

with open('compile_commands.json') as f:
    db = json.load(f)

for entry in db:
    if 'arguments' in entry:
        entry['arguments'] = [a for a in entry['arguments'] if not should_remove(a)]
    elif 'command' in entry:
        args = shlex.split(entry['command'])
        args = [a for a in args if not should_remove(a)]
        entry['command'] = ' '.join(shlex.quote(a) for a in args)

with open('compile_commands.json', 'w') as f:
    json.dump(db, f, indent=2)

print(f"Processed {len(db)} entries in compile_commands.json")
```

### `.claude/scripts/commit-dev.sh`

**変更箇所: `git add` の直前に clang-format チェックを追加（64行目付近）**

```bash
# Before:
git add -u
# 新規ファイルも追加（git add -u は未追跡ファイルをステージしないため）
git add source/ assets/ 2>/dev/null || true

# After:
# clang-format チェック（インストールされている場合のみ）
if command -v clang-format >/dev/null 2>&1; then
  MODIFIED_CPP=$(git diff --name-only HEAD -- '*.cpp' '*.c' '*.h' '*.hpp' 2>/dev/null || true)
  if [ -n "$MODIFIED_CPP" ]; then
    echo "[commit-dev] clang-format check..."
    if ! echo "$MODIFIED_CPP" | xargs clang-format --dry-run --Werror -style=file 2>&1; then
      echo "ERROR: clang-format violations. Fix with: echo \"<files>\" | xargs clang-format -i -style=file" >&2
      exit 1
    fi
    echo "[commit-dev] clang-format OK"
  fi
else
  echo "[commit-dev] clang-format not found in PATH — skipping format check" >&2
fi

git add -u
# 新規ファイルも追加（git add -u は未追跡ファイルをステージしないため）
git add source/ assets/ 2>/dev/null || true
git add .github/ .clang-format .clang-tidy 2>/dev/null || true
```

## 4. 各機能の詳細設計

### FR-7/FR-8: `-Wextra` と `-Werror`

`ifdef CI` は GitHub Actions が `CI=true` を環境変数として自動注入することを利用する。ローカルの make には CI 変数が存在しないため `-Werror` は付かない。

`-Wextra` で新たに出る可能性がある警告カテゴリ：
- `-Wunused-parameter`: 未使用引数 → `(void)param;` か `[[maybe_unused]]`
- `-Wmissing-field-initializers`: 構造体の部分初期化 → `= {}` で全ゼロ初期化
- `-Wsign-compare`: signed/unsigned 比較 → `static_cast<int>` で揃える

### FR-5: clang-tidy 試行の判定基準

CI ログを確認し、以下を達成できたら必須ゲートに昇格（`continue-on-error: false` に変更）：
- compile_commands.json が生成されている
- clang-tidy が SDK ヘッダの警告を出力していない（HeaderFilterRegex が効いている）
- 実際のバグ指摘が source/ のコードに対して出ている

上記を 2 PR 連続で確認できたら昇格とする。

## 6. 実装順序

### ステップ 1: `.clang-format` の確定とスタイル一括適用

1. `.clang-format` を作成
2. `clang-format -i -style=file $(git ls-files '*.cpp' '*.h')` で全ファイルに適用
3. 差分を確認してスタイルが妥当か検証（必要なら設定調整）
4. **ビルド確認**: スタイル変更のみなのでビルドへの影響はないはずだが念のため実行
5. 専用コミット「Apply clang-format to all source files」

### ステップ 2: Makefile `-Wextra` 追加と警告修正

1. `Makefile` の `CFLAGS` に `-Wextra` を追加
2. ローカルでビルドして警告一覧を確認
3. 警告を修正（`-Wunused-parameter`、`-Wsign-compare` 等）
4. **ビルド確認**（クリーンビルドで警告 0 を確認）
5. コミット

### ステップ 3: GitHub Actions CI ワークフロー（format + build ジョブ）

1. `.github/workflows/ci.yml` を作成（`clang-tidy` ジョブはまだ除く）
2. `dev` ブランチへコミット・プッシュ
3. `main` への PR を作成して CI が動くか確認
4. `format` ジョブ・`build` ジョブが green になるまで修正

### ステップ 4: cppcheck の確認

- ステップ 3 の `build` ジョブ内に cppcheck ステップが含まれているので、その出力を確認
- false positive が多い場合は `--suppress` を追加調整

### ステップ 5: commit-dev.sh 更新

1. clang-format チェックブロックを追加
2. `git add` 対象に `.github/`、`.clang-format`、`.clang-tidy` を追加
3. ローカルで動作確認

### ステップ 6: clang-tidy 試行

1. `.clang-tidy` を作成
2. `strip_arm_flags.py` を追加
3. `ci.yml` に `clang-tidy` ジョブを追加（`continue-on-error: true`）
4. PR を作成して CI ログを確認
5. 判定基準を満たしたら `continue-on-error: false` に変更

### ステップ 7: CodeQL

1. `.github/workflows/codeql.yml` を作成
2. PR を作成して Security タブにアラートが出るか確認

### ステップ 8: レビュースキル・ワークフロー設定の更新

1. `.claude/skills/cpp-3ds-reviewer/SKILL.md` に clang-format チェックをステップ 0 として追加
2. `.claude/harness-config.json` の Review ステップメッセージを更新
