# システム設計書 - Phase 5 Font Fallback
**RDD:** `RDD-2026-04-11-1400.md`
**作成日:** 2026-04-11

## 1. 全体方針

### 主要な設計判断
1. **フォント管理は ui.cpp に閉じる**
   - 新ファイルを作らず、既存の描画層である `ui.cpp` の static 変数として
     `C2D_Font` を保持する。`drawText` / `measureStr` のローカル関数を修正
     するだけで完結し、ヘッダ (`ui.h`) への影響はゼロ。

2. **`C2D_Text` のキャッシュは一切行わない**
   - coding-patterns #7 に従い、分割描画した区間ごとに毎フレーム
     `C2D_TextParse` / `C2D_TextFontParse` を呼ぶ。

3. **区間分割は「連続した同一フォント種別のチャンク」単位**
   - UTF-8 走査で 1 文字ごとに判定するが、描画単位は「同じフォントで連続する
     区間」にまとめる。区間ごとに `C2D_Text` を作って描画 → 次の区間のために
     x オフセットを `t.width * sx` だけ進める。

4. **フォント未ロード時は既存動作を維持**
   - `fallbackFont == nullptr` のときは、既存の `C2D_TextParse` パスを
     そのまま通す（分割しない）。`？` 表示は変わらないが、アプリは正常動作。

5. **`mkbcfnt` はビルド時に呼ぶ**
   - Makefile に `romfs/fallback.bcfnt` を生成するターゲットを追加。
     `assets/fonts/*.ttf` と whitelist から変換。依存関係管理は make の
     組み込み機能で済む。

### 非スコープ（DESIGN でも扱わない）
- フォントの動的リロード
- 複数ランタイム言語切替
- `wrapText` 内の推定幅計算（`utf8CharPx`）の精度向上 — 現行ヒューリスティクスで十分

#### 新規ヘルパー（drawText の上に追加）
```cpp
// UTF-8 1文字を読み、コードポイントとバイト長を返す。
// 不正バイトは U+FFFD とみなし、1 バイト進める。
static uint32_t utf8Decode(const char* s, int& outBytes) {
    unsigned char c = (unsigned char)s[0];
    if (c < 0x80) { outBytes = 1; return c; }
    if ((c & 0xE0) == 0xC0) {
        outBytes = 2;
        return ((c & 0x1F) << 6) | ((unsigned char)s[1] & 0x3F);
    }
    if ((c & 0xF0) == 0xE0) {
        outBytes = 3;
        return ((c & 0x0F) << 12)
             | (((unsigned char)s[1] & 0x3F) << 6)
             |  ((unsigned char)s[2] & 0x3F);
    }
    if ((c & 0xF8) == 0xF0) {
        outBytes = 4;
        return ((c & 0x07) << 18)
             | (((unsigned char)s[1] & 0x3F) << 12)
             | (((unsigned char)s[2] & 0x3F) << 6)
             |  ((unsigned char)s[3] & 0x3F);
    }
    outBytes = 1;
    return 0xFFFD;
}

// システムフォントで描画可能か（= alterCharIndex に落ちないか）
static bool systemHasGlyph(uint32_t cp) {
    CFNT_s* sys = fontGetSystemFont();
    int alter = sys->finf.alterCharIndex;
    int idx   = fontGlyphIndexFromCodePoint(sys, cp);
    return idx != alter;
}
```

#### `drawText` の置換
既存:
```cpp
static void drawText(const char* str, float x, float y, float z,
                     float sx, float sy, u32 color) {
    C2D_Text text;
    C2D_TextParse(&text, textBuf, str);
    C2D_TextOptimize(&text);
    C2D_DrawText(&text, C2D_WithColor, x, y, z, sx, sy, color);
}
```

変更後:
```cpp
// システムフォントで欠損しているコードポイントを補完フォントで描画する。
// 補完フォント未ロード時は単純に全文字をシステムフォントで描画。
static void drawText(const char* str, float x, float y, float z,
                     float sx, float sy, u32 color) {
    if (!fallbackFont) {
        // フォールバック未使用: 既存パスと完全一致
        C2D_Text text;
        C2D_TextParse(&text, textBuf, str);
        C2D_TextOptimize(&text);
        C2D_DrawText(&text, C2D_WithColor, x, y, z, sx, sy, color);
        return;
    }

    // UTF-8 を走査し、「システム区間」と「補完区間」に分けて連続チャンクを構築
    float cursorX = x;
    size_t n = strlen(str);
    size_t i = 0;
    while (i < n) {
        // 現在位置の文字種（system or fallback）を判定
        int bytes;
        uint32_t cp = utf8Decode(str + i, bytes);
        bool useSystem = systemHasGlyph(cp);

        // 同種が連続する最長区間 [i, j) を伸ばす
        size_t j = i + bytes;
        while (j < n) {
            int nb;
            uint32_t ncp = utf8Decode(str + j, nb);
            bool ns = systemHasGlyph(ncp);
            if (ns != useSystem) break;
            j += nb;
        }

        // 区間文字列を作って描画
        // （textBuf はフレーム先頭で Clear 済み。各区間の C2D_Text は使い捨て）
        std::string chunk(str + i, j - i);
        C2D_Text text;
        if (useSystem) {
            C2D_TextParse(&text, textBuf, chunk.c_str());
        } else {
            C2D_TextFontParse(&text, fallbackFont, textBuf, chunk.c_str());
        }
        C2D_TextOptimize(&text);
        C2D_DrawText(&text, C2D_WithColor, cursorX, y, z, sx, sy, color);

        cursorX += text.width * sx;
        i = j;
    }
}
```

変更後:
```cpp
// drawText と同じ区間分割ロジックで幅を合算する。
static float measureStr(const char* str, float scale) {
    C2D_TextBufClear(measureBuf);

    if (!fallbackFont) {
        C2D_Text t;
        C2D_TextParse(&t, measureBuf, str);
        return t.width * scale;
    }

    float total = 0.0f;
    size_t n = strlen(str);
    size_t i = 0;
    while (i < n) {
        int bytes;
        uint32_t cp = utf8Decode(str + i, bytes);
        bool useSystem = systemHasGlyph(cp);

        size_t j = i + bytes;
        while (j < n) {
            int nb;
            uint32_t ncp = utf8Decode(str + j, nb);
            if (systemHasGlyph(ncp) != useSystem) break;
            j += nb;
        }

        std::string chunk(str + i, j - i);
        C2D_Text t;
        if (useSystem) {
            C2D_TextParse(&t, measureBuf, chunk.c_str());
        } else {
            C2D_TextFontParse(&t, fallbackFont, measureBuf, chunk.c_str());
        }
        total += t.width;
        i = j;
    }
    return total * scale;
}
```

```make
# --- 既存 ---
ROMFS		:=	romfs

# --- 追加: フォント生成 ---
FALLBACK_FONT := $(CURDIR)/romfs/fallback.bcfnt
FONT_TTF      := $(CURDIR)/assets/fonts/NotoSansSymbols2-Regular.ttf
FONT_WHITELIST := $(CURDIR)/assets/fonts/symbols_whitelist.txt

$(FALLBACK_FONT): $(FONT_TTF) $(FONT_WHITELIST)
	@echo "mkbcfnt: generating $(notdir $@)"
	@mkbcfnt -w $(FONT_WHITELIST) -o $@ $(FONT_TTF)

# `all:` に依存を追加（第一パスのルールセクション内）
all: $(FALLBACK_FONT) $(BUILD)

# clean で削除
clean:
	@echo clean ...
	@rm -fr $(BUILD) $(TARGET).3dsx $(OUTPUT).smdh $(TARGET).elf $(FALLBACK_FONT)
```

注意: Makefile はトップレベルと BUILD 配下の二段構成。フォント生成は
トップレベル側（`ifneq ($(BUILD),$(notdir $(CURDIR)))` ブランチ）に追加する。

## 4. 各機能の詳細設計

### FR-1: フォント読込
- `uiInit()` 内で `C2D_FontLoad("romfs:/fallback.bcfnt")` を呼ぶ
- 戻り値 nullptr は正常系として扱う（静かに機能を無効化）
- `romfsInit()` の成否は libctru が管理するが、3dsx では常に成功するので
  本実装では戻り値チェック不要

### FR-2: 自動フォールバック描画
疑似コード:
```
入力: UTF-8 文字列 str, 描画座標 (x,y)
i = 0, cursorX = x
while i < len(str):
    cp, bytes = decode_utf8(str[i:])
    useSystem = system_has_glyph(cp)
    j = i + bytes
    # 同種の連続を伸ばす
    while j < len(str):
        ncp, nb = decode_utf8(str[j:])
        if system_has_glyph(ncp) != useSystem: break
        j += nb
    chunk = str[i:j]
    text = Parse(chunk, useSystem ? systemFont : fallbackFont)
    DrawText(text, cursorX, y, ...)
    cursorX += text.width * sx
    i = j
```

パフォーマンス見積もり:
- 典型的な記事 1 行（~50 文字）でシステム区間 1 つ＋小記号区間 1 つ程度
- 1 フレームあたりの区間数は全画面で 100 未満 → `C2D_TextParse` 呼出数
  は現状比 2〜3 倍。3DS 上でもフレームレート影響は無視できる見込み
  （重いのは `C2D_DrawText` の GPU 送信、パース自体は軽い）

### FR-3: メトリクス整合
- `mkbcfnt` のデフォルトサイズは元 TTF の推奨サイズを使用
- `-s 22` 等で明示指定可能だが、まずはデフォルトで変換 → 実機で
  システムフォント (height=30, baseline=25) との整合を確認
- ずれていたら `-s` で調整（実装後のテスト段階で対応）

### FR-4: 三段フォールバック
`systemHasGlyph` で false → 補完フォントで描画。
補完フォントにも無い場合は `C2D_TextFontParse` が内部で補完フォントの
`alterCharIndex`（= 補完フォントの `？` or 空白）を使う。
これは既存動作と同等の「欠損文字の劣化表示」なので RDD 通り。

### FR-5: Whitelist
`assets/fonts/symbols_whitelist.txt` に列挙。初期 ~60 文字。
実機テストで不足したら追加してビルド。

### FR-6: ビルド統合
`make` 時に `$(FALLBACK_FONT)` が生成されてから `$(BUILD)` に入る。
TTF/whitelist を更新すると自動再生成。`make clean` で削除。

## 5. 実装順序

各段階の末尾でビルド確認を行う。どこかで止まれば前段階に戻って原因切り分け。

1. **フォント素材配置**
   - `assets/fonts/` 作成
   - Noto Sans Symbols 2 TTF をダウンロード配置
   - `symbols_whitelist.txt` 作成（~60 文字）
   - `FONT_LICENSE.txt` 配置
   - → この段階では Makefile 未変更、ビルド確認なし

2. **Makefile: mkbcfnt ルール追加**
   - `all: $(FALLBACK_FONT) $(BUILD)` と生成ルール追加
   - `clean` 拡張
   - → `make` 実行 → `romfs/fallback.bcfnt` が生成されることを確認
   - → `.gitignore` に追加

3. **main.cpp: romfsInit 追加**
   - `gfxInitDefault()` の直後に 1 行追加
   - → `make` 実行 → ビルド通ること確認（動作は変わらない）

4. **ui.cpp: フォント読込のみ**
   - `fallbackFont` static 変数追加
   - `uiInit` / `uiExit` を拡張
   - `drawText` / `measureStr` はまだ変更しない
   - → `make` → 起動して既存画面が壊れないこと確認
   - → stderr ログで読込成否を確認

5. **ui.cpp: フォールバック描画ロジック**
   - `utf8Decode`, `systemHasGlyph` ヘルパー追加
   - `drawText` を区間分割版に置換
   - `measureStr` も区間分割版に置換
   - → `make clean && make`（coding-patterns #10 は今回適用外だが念のため）
   - → 起動して既存画面（フィード一覧・記事一覧・記事本文）が壊れないこと確認
   - → ITMedia 等の記号を含む記事を開いて記号が表示されることを確認

6. **実機フィードバック反映**
   - 記号がまだ `？` の場合: whitelist に追加 → 再ビルド
   - ベースラインずれ: `mkbcfnt -s <N>` を調整
