# システム設計書 - Phase 6.5 Readability 本格移植
**RDD:** `RDD-2026-04-15-1200.md`
**作成日:** 2026-04-15

## 1. 全体方針

- 現行 `html_strip.cpp` の `extractContent()` を破棄し、新ファイル `readability.cpp/.h` に置換。`stripHtml()` は RSS description 整形でも使うため現行ファイルに残す
- 3 フェーズ構成（pre-clean → Readability → post-clean）を明示的な関数分割で実装
- Readability アルゴリズムは Mozilla Readability.js の JS コードを参照しつつ C++ / lexbor DOM API で再実装
- スコアリングは CJK 文字を 1 文字としてカウントする UTF-8 文字数ベース（現行のバイト数ベースは日本語で過剰評価される）
- 進捗コールバックは関数ポインタ方式で `extractContent()` の引数に追加。既存呼び出し側は nullptr 互換を保つ
- 既存公開 API `extractContent(html, url)` は維持しつつ、3 引数目にオプションコールバックを追加
- テスト用サンプル HTML は `tests/samples/` に保存。検証は 3DS 実機で `extract_log.txt` と記事画面目視で実施（ホスト側ユニットテストは lexbor ホストビルド手配が過大のため本フェーズ外）

## 2. 新規ファイル

### `source/readability.h`

```cpp
#pragma once
#include <string>

// 抽出ステージ（進捗通知用）
enum class ReadabilityStage {
    Parse,      // lexbor パース開始
    PreClean,   // pre-clean 完了
    Score,      // スコアリング完了
    PostClean,  // post-clean 完了
    Done        // テキスト化完了
};

// 進捗コールバック。user は呼び出し側から任意に渡される。
// percent は 0〜100 の概算値（各ステージの完了時点で更新）。
using ReadabilityProgressCb = void(*)(int percent,
                                      ReadabilityStage stage,
                                      void* user);

// 本文抽出。失敗時は stripHtml(html) を返す。
// url が空でなければ sdmc:/3ds/rssreader/extract_log.txt にログ。
// cb が非 nullptr なら各ステージ完了時に呼ばれる。
std::string extractContent(const std::string& html,
                           const std::string& url = "",
                           ReadabilityProgressCb cb = nullptr,
                           void* cb_user = nullptr);
```

## 4. 各機能の詳細設計

### FR-1〜FR-3: 3 フェーズパイプライン

`extractContent()` のメイン流れ:

```
1. 入力サイズを MAX_HTML_BYTES に切り詰め
2. lxb_html_document_create / parse
   - 失敗 → stripHtml フォールバック、ログ "FALLBACK/parse_err"
3. notify(Parse, 10%)
4. body 取得（失敗 → フォールバック）
5. preClean(body)
   - UNLIKELY_TAGS をサブツリーごと削除
   - class/id が NOISE_KW に一致する要素をサブツリーごと削除
   - simple_walk でノードを収集 → 後処理で一括 remove
6. notify(PreClean, 30%)
7. findTopCandidate(body):
   - body 配下を simple_walk
   - タグが p/pre/td/li の各ノードに対して:
       a. nodeScore = 1 + split(",", textContent).length + min(floor(charCount/100), 3)
       b. linkDensity = linkCharCount / totalCharCount （リンク密度）
       c. finalScore = nodeScore * (1 - linkDensity) * (1 - classPenalty)
       d. 親（最大 3 階層）にスコアを伝播（親=score, 祖父=score/2, 曾祖父=score/3）
   - candidates 配列に保持。score 降順で最高を topCandidate として選定
   - topCandidate なし → body 全体
8. notify(Score, 60%)
9. mergeSiblings(topCandidate, candidates, threshold):
   - 親ノードの子を走査
   - 各兄弟の score が topCandidate.score * 0.2 以上 または class 類似なら
     新しい結合ノード（または本文候補として採用）に含める
   - 簡易実装: 採用する兄弟ノードを vector に集め、serializeText で連結
10. notify(PostClean, 80%) の前に postClean(採用サブツリー):
    - POST_NOISE_KW に一致するクラス/id のノードを削除
    - 空の <p> や極端に短い <div> を削除（オプション）
11. notify(PostClean, 80%)
12. serializeText(採用サブツリー) → 結果文字列
    - BLOCK_TAGS では改行挿入
    - SKIP_SUBTREE_TAGS（pre 等）は子を走査せずテキストそのまま
13. 連続改行正規化（最大 2 連続）
14. 空なら stripHtml フォールバック、ログ "FALLBACK/empty"
15. logEntry(url, "OK", winnerTag, topScore, length)
16. notify(Done, 100%)
17. document destroy して return
```

### FR-4: CJK 対応スコアリング

`countCharsUtf8`:

```cpp
static size_t countCharsUtf8(const lxb_char_t* s, size_t byteLen) {
    size_t count = 0;
    for (size_t i = 0; i < byteLen; ) {
        unsigned char c = s[i];
        if      (c < 0x80)         { i += 1; }
        else if ((c & 0xE0)==0xC0) { i += 2; }
        else if ((c & 0xF0)==0xE0) { i += 3; } // CJK はほぼここ
        else if ((c & 0xF8)==0xF0) { i += 4; }
        else                       { i += 1; } // 不正バイト
        ++count;
    }
    return count;
}
```

全てのスコア計算（`nodeScore`, `linkDensity` の分子分母）でバイト数ではなく文字数を使う。

### FR-5: 進捗コールバック

`ReadabilityProgressCb` は関数ポインタ型（`std::function` はサイズ大）。`extractContent()` は受け取った `cb` を各ステージ終了時に呼び出すだけ。

本フェーズでは `article.cpp` は `cb=nullptr` で呼び出し、実 UI 配線は行わない。

### FR-6: 現行コード削除

差分は「3. 既存ファイルの変更」参照。`html_strip.cpp` の行 183〜543 を削除。`Makefile` の SOURCES に `source/readability.cpp` を含めるための対応は不要（`SOURCES := source` で自動拾われる）。

### NFR-2: ベースライン計測

新エンジン実装 **前** に現行 `extractContent()` のタイミング計測を追加:
- `article.cpp` で `osGetTime()` を呼び出し前後で記録
- `extract_log.txt` に `elapsed_ms` を追記するよう現行ログ関数を一時拡張
- 主要 3 サイト × 各 3 記事で計測、`HANDOFF/phases.md` もしくは本フェーズ用メモに記録
- 計測後、新エンジン実装に入る
